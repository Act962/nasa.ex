# N.A.S.A — Contexto de Vendas para Claude

> **Para quem é este documento.** Este é o briefing oficial pra qualquer Chat do Claude (claude.ai, API, agentes customizados) ajudar a vender a N.A.S.A. Carregue-o como contexto e o Claude entende identidade, ecossistema, diferenciais, personas, modelo de negócio e tem templates de pitch prontos.
>
> **Última atualização:** 2026-06-02 (após PR #79/#80/#81).
> **Mantenedor:** Wey + Claude.

---

## 0. Como usar este documento

**Instruções para o Claude:**

1. Antes de propor um pitch, **identifique o público** (founder solo? gestor comercial? agência? mentor digital? franquia? indústria com SDR?). Cada pilar do ecossistema brilha em um contexto diferente — não despeje tudo de uma vez.
2. **Não venda "CRM"**. NASA é mais que CRM — é o **sistema operacional do dia-a-dia comercial e operacional**. Use o vocabulário do § 2 abaixo.
3. Quando perguntarem "isso já existe?" — diga **o que está em produção hoje** (§ 4 → coluna "Status"). Só prometa futuros se a pergunta for sobre roadmap.
4. **Diferenciais sempre antes de features.** O usuário escuta "agenda" e pensa em Calendly. Você precisa explicar PRIMEIRO o porquê (§ 5) pra depois listar features.
5. **Stars (★)** é nossa moeda interna. Toda explicação de pricing passa por ela (§ 7).
6. Mantenha tom: **espacial sem cringe, técnico sem jargão vazio, pragmático sem ser cinza.**

## 1. Identidade & Posicionamento

### 1.1 Slogan — 3 opções

Cada um pra um cenário. Comece pelo **A** e teste reação.

| # | Slogan | Quando usar |
|-|-|-|
| **A** | **"O Sistema Operacional do seu time comercial e de atendimento"** | Pitch curto, audiência cética. Posiciona escala (SO ≠ ferramentinha) sem prometer demais. |
| **B** | **"O ecossistema operacional de quem vende, atende e entrega no mesmo time"** | Pitch médio, audiência consultiva (agências, mentores). Captura a triple-job. |
| **C** | **"Onde processo comercial, atendimento e equipe rodam juntos — não em 7 abas"** | Pitch dor-primeiro, audiência que já sofre com stack fragmentado (RD + Whaticket + Asana + ClickFunnels + Google Calendar + Loom + Zapier). |

**Por que NÃO "Maior ecossistema de soluções para gestão de processos comerciais e atendimento do mundo":**
- 16 palavras antes do verbo. O ouvido desiste no "soluções para gestão".
- "Maior" exige prova. Você ainda não tem volume público.
- "Do mundo" parece bluff.

**Variante curta defensável:**
> "NASA. O OS do seu negócio."

### 1.2 Manifesto (180 caracteres)

> Toda equipe que vende e atende vive entre 7 abas. NASA junta tudo num só ecossistema — onde o dado anda com o lead, a IA conhece o contexto e o time entrega sem repetir trabalho.

### 1.3 Jobs to be Done (que NASA realmente cumpre)

Listados em ordem de força. Cada um vira um ângulo de pitch.

1. **"Quero que minha operação rode sem eu ter que olhar 6 ferramentas e cruzar planilha"** → Hub unificado (CRM + Atendimento + Tarefas + Agendas + Financeiro + Cursos)
2. **"Quero que minha equipe NÃO esqueça lead, follow-up, contrato, comprovante, formulário ou tarefa"** → Automações com IA + Auto Agentes
3. **"Quero que cada lead receba um atendimento personalizado, mesmo a 3h da manhã"** → Astro (IA do NASA) + WhatsApp Bot + Chatbot in-chat
4. **"Quero vender meu conhecimento (curso, mentoria, comunidade) sem precisar de Hotmart + Teachable + Discord + Zoom + Mailchimp"** → NASA Route
5. **"Quero criar páginas, propostas, contratos, formulários, agendas sem precisar de dev"** → Forge, Linnker, Form, Agenda, NBox
6. **"Quero que minha equipe trabalhe próxima mesmo remota"** → Space Station + In-Chat + Tracking Chat
7. **"Quero relatórios de verdade — quem produz, onde trava, quanto custa"** → Insights + Activity Logs + Stars Economy

                        │  Cmd+K / WhatsApp   │

┌──────┐ ┌────────┐ ┌──────┐  ┌──────────┐ ┌────────┐ ┌──────┐ ┌──────┐

│COMER-│ │  ÇÕES  │ │CAÇÃO │  │ ANEXOS   │ │MENTO   │ │BORAÇÃO│ │ MIA  │
│CIAL  │ │  & IA  │ │      │  │  (FORGE) │ │(ROUTE) │ │      │ │STARS │
└──────┘ └────────┘ └──────┘  └──────────┘ └────────┘ └──────┘ └──────┘
                                   │
                            ┌──────┴──────┐

### Quem é o eixo: **Astro**

A IA que conhece **tudo**: lead, conversa, agenda, contrato, financeiro, time. Acessível via:
- Cmd+K em qualquer tela (Astro Explorer)
- WhatsApp pessoal (Astro Bot — PR #79)
- Embutido em workflows (Modo Agente IA)
- Voz (em breve)

Não é "assistente de IA" — é **o jeito de operar o sistema**.

## 5. Diferenciais Competitivos (use ANTES das features)

### 5.1 Por que NASA ≠ "mais um CRM"

| Concorrente típico | O que ele faz | O que NASA tem a mais |
|-|-|-|
| **RD Station / Pipedrive / HubSpot** | Funil + e-mail marketing + automação básica | Tudo isso + IA conversacional integrada + WhatsApp nativo (uazapi e Meta) + plataforma de cursos (Route) + economia de créditos transparente (Stars) + colaboração 2D (Space Station) |
| **Whaticket / Octadesk / Take Blip** | Atendimento WhatsApp multi-instância | Tudo isso + funil + agente IA com tools de mutação + auto-agent autônomo + chat interno (In-Chat) + workflows complexos com validação |
| **Hotmart / Kiwify / Eduzz** | Vender curso + checkout | Tudo isso + base de leads atrelada + e-mail pós-compra customizável + comunidade + cobrança via Stars (sem perda em conversão) + revenue split nativo |
| **Calendly / Cal.com** | Agenda pública | Tudo isso + lead criado automaticamente + workflow disparado por agendamento + recurrence + status (reivindicar/denunciar) integrado ao CRM |
| **Notion / ClickUp / Asana** | Tasks + Docs | Workspace embutido + automações cross-app + IA com contexto de venda/atendimento |
| **Zapier / Make** | Conectar apps | Workflows internos onde TODOS os apps já são do NASA — sem rate limit externo, sem licença por action, sem ETL |
| **Linktree / Beacons** | Página de bio | Linnker + página pública + integrado aos cliques no workflow |
| **Gather.town / Sococo** | Sala virtual 2D | Space Station + dentro do CRM (gestor vê quem tá próximo de quem) |
| **Loom / Vidyard** | Vídeo async | NASA Route + vídeos hospedados + analytics + conversão pra venda |

### 5.2 Diferencial estrutural (não-óbvio)

1. **Tudo é um lead.** Curso vendido vira lead. Reunião agendada vira lead. Form preenchido vira lead. Comentário em proposta vira lead. **Isso permite que o mesmo workflow trate qualquer entrada — você programa uma vez, dispara em qualquer canal.**

2. **Astro conhece tudo.** A IA tem acesso ao banco inteiro (com escopo por permissão). Você fala "qual lead tá há mais tempo no status 'Aguardando Pagamento'?" — ele responde com nome, tracking, dias parado e botão de ação inline. Concorrentes ou (a) não têm IA, ou (b) têm IA isolada num módulo.

3. **Stars como contract de pricing.** Você compra Stars, gasta como quiser. Sem feature flag, sem "esse plano não tem isso", sem upgrade forçado. Modelo radical de UX. Owner vê **exatamente** o que custou cada operação.

4. **Workflows com IA decisora.** AI_DECISION node não é "envia pro ChatGPT". É **estado dentro do fluxo** — workflow pausa, IA analisa contexto + memória, decide próximo branch. Outros têm "AI Action" — aqui é "AI Branch".

5. **Multi-tenant nativo profundo.** Não é "deploy separado por cliente". É um único banco com isolamento por `organizationId` em **toda** query. Quando você vê WhatsApp banido só na sua org, sua infraestrutura segue saudável.

6. **Hot-reload schema versioning** (`SCHEMA_VERSION`). Engine interna dá graceful degrade quando migration não foi aplicada — clientes em produção não quebram durante deploy.

## 9. FAQs comuns + Objeções

### "É mais um CRM?"
> "Não. CRM é uma das 6 partes. NASA também é WhatsApp Business + plataforma de cursos + sala virtual + ferramenta de proposta + agenda pública + chat interno. **Tudo no mesmo banco, falando entre si.** Concorrente que tem 1 das 6 cobra mais que nós, que temos as 6."

### "Tem integração com [X ferramenta]?"
> Já temos: Meta Ads, Google Calendar, Stripe, Asaas, RD Station, Pipedrive, TikTok, Instagram, LinkedIn, Kwai, Resend. Falta? HTTP_REQUEST node em workflow chama qualquer API.

### "Vocês não vão escalar — não conheço a empresa"
> "A engine roda em produção em Vercel + Neon + Inngest. Multi-tenant nativo, hot-reload, observabilidade real. Open posture: você quer ver a arquitetura? Mostro o GitHub agora."

### "WhatsApp não-oficial vai banir"
> "Sim, vai. Por isso temos duas tiers: uazapi (Earth, NASA absorve risco) e Meta Cloud API oficial (Suite/Constellation, cliente paga ~R$0,025/conv). Quando uazapi bana, o In-Chat fallback ativa — link público no browser pra lead continuar. **Você não perde nenhuma conversa em curso**, e sai do ban em 24h migrando pra outra instância ou subindo pra Cloud."

### "Quanto custa em comparação com [stack atual]?"
> Faça a conta com o cliente. Ex:
> - RD Station Pro: R$ 1.499/mês
> - Whaticket: R$ 399/mês
> - Hotmart taxa: 9.9% sobre vendas
> - Calendly: $12/user/mês
> - Asana: $11/user/mês
> Total para uma equipe de 5: ~R$ 2.500/mês fixo + 9.9% das vendas.
>
> NASA Constellation: 50.000★ ≈ R$ 1.999/mês + 5% sobre cursos. Plus você tem cobrança transparente (vê onde foi cada R$).

### "Como faço a migração?"
> "Importadores nativos pra RD/Pipedrive/XLSX. Para automações, você aplica Padrões NASA (5 prontos) e ajusta com nosso copy/paste de workflow. Acompanhamento na primeira semana incluído."

### "Tem white-label?"
> Roadmap. Hoje você usa domínio NASA. Pra contratos enterprise, conversamos sobre Suite + setup customizado.

### "E se eu perder os dados?"
> "Banco Postgres com backups automáticos (Neon snapshots). Export por org via API — você pode tirar tudo num zip se quiser. Auditoria StarTransaction + Activity Log preserva todas as ações irreversíveis."

### "IA usa meu dado pra treinar?"
> Não. Astro usa **seu** banco em tempo de consulta. Não há fine-tuning com dados de cliente. Provedores LLM (Anthropic, OpenAI, Google) tratam queries como ephemeral conforme TOS deles.
