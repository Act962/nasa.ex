# Etapa 6 — Componentes UI

## Objetivo

Criar 3 componentes dentro de `src/features/fiscal/components/` e integrá-los na UI do Forge existente:

1. **`fiscal-profile-form.tsx`** — formulário de configuração fiscal da org (dentro de ForgeSettings)
2. **`issue-invoice-dialog.tsx`** — diálogo de emissão com dados do tomador e pré-flight de erros
3. **`fiscal-invoice-card.tsx`** — card de status da nota no detalhe do contrato

## `src/features/fiscal/components/fiscal-profile-form.tsx`

Formulário com React Hook Form + Zod. Usa `useFiscalProfile()` para carregar e `useUpsertFiscalProfile()` para salvar.

Campos:
- CNPJ do prestador (máscara XX.XXX.XXX/XXXX-XX)
- Razão social
- Inscrição municipal
- Código IBGE do município (7 dígitos)
- Regime: Optante Simples Nacional (toggle), regime especial de tributação (opcional)
- Endereço: logradouro, número, complemento, bairro, CEP, UF
- Defaults do serviço: item lista de serviço (LC 116), alíquota ISS (%), ISS retido (toggle), discriminação padrão
- Ambiente: HOMOLOGACAO | PRODUCAO (select)
- Município suportado pela Focus (checkbox)

Ao salvar, exibe:
- Toast de sucesso + badge "Empresa cadastrada na Focus ✓" se `focusEmpresaRegistered = true`
- Toast de aviso "Empresa não encontrada na Focus — cadastre o certificado A1 no painel" se `focusEmpresaRegistered = false`

## `src/features/fiscal/components/issue-invoice-dialog.tsx`

Dialog (Radix UI `Dialog`) aberto pelo botão "Emitir nota fiscal" no card/detalhe do contrato.

```
Props: { contractId: string; contractNumber: number; contractValue: string; clientData: ClientData | null }
```

### Campos do formulário

**Dados do tomador** (pré-preenchidos do `clientData` do contrato, editáveis):
- Tipo de tomador: PF | PJ (radio)
- **PJ**: CNPJ, Razão Social, e-mail, logradouro, número, complemento, bairro, código IBGE município, UF, CEP
- **PF**: CPF, Nome completo, e-mail

**Dados do serviço**:
- Competência (mês/ano, default = mês atual — usar `<input type="month">`)
- Discriminação do serviço (textarea, pré-preenchido com default do perfil)

**Resumo antes de emitir**:
- Valor: R$ X (do contrato)
- Alíquota ISS: X% (do perfil)
- ISS retido: Sim/Não
- Ambiente: HOMOLOGAÇÃO / PRODUÇÃO

### Comportamento

1. Ao abrir: executa pré-flight via `validateBeforeEmit` (client-side, mesma lógica do servidor) → exibe erros em bloco vermelho se houver. Botão "Emitir" fica desabilitado enquanto há erros.
2. Ao submeter: `useIssueFiscalInvoice().mutate(...)` com `onSuccess` mostrando toast "Nota fiscal enviada para processamento" e fechando o dialog.
3. Se `status = "AUTORIZADO"` na resposta: toast "Nota fiscal autorizada!" (município síncrono).
4. Se `status = "ERRO"`: toast de erro com a mensagem da Focus.

## `src/features/fiscal/components/fiscal-invoice-card.tsx`

```
Props: { contractId: string }
```

Usa `useFiscalInvoicesByContract(contractId)`.

### Layout

```
┌─────────────────────────────────────────────┐
│ Nota Fiscal  [badge: PROCESSANDO|AUTORIZADO|ERRO|CANCELADO]
│                                             │
│ NFS-e #1234  competência 05/2025           │
│ Valor: R$ 5.000,00                         │
│                                             │
│ [PDF] [XML] [Atualizar status] [Cancelar]  │
│                                             │
│ Mensagem de erro (se ERRO)                 │
└─────────────────────────────────────────────┘
```

### Badges de status

| Status | Cor |
|--------|-----|
| PROCESSANDO | Amarelo (pendente) |
| AUTORIZADO | Verde |
| ERRO | Vermelho |
| CANCELADO | Cinza |

### Botões contextuais

- **PDF** (link `urlDanfse`, só se AUTORIZADO)
- **XML** (link `caminhoXmlStorage ?? caminhoXmlFocus`, só se AUTORIZADO)
- **Atualizar status** (só se PROCESSANDO) — `useRefreshFiscalInvoiceStatus()`
- **Cancelar nota** (só se AUTORIZADO) — abre mini-dialog de confirmação com campo `justificativa` (mínimo 15 chars)

Se `invoices.length === 0`:
```
[Emitir nota fiscal]  → abre IssueInvoiceDialog
```

Se já existe invoice ativa (PROCESSANDO|AUTORIZADO): botão "Emitir nota fiscal" fica oculto (guard visual, além do guard de servidor).

## Validação desta etapa

1. Preencher perfil fiscal via Settings → Forge → Fiscal — verificar `FiscalCompanyProfile` salvo no banco.
2. Acessar um contrato → botão "Emitir nota fiscal" aparece.
3. Abrir diálogo → pré-flight mostra erros se perfil incompleto.
4. Emitir nota em homologação → card mostra `PROCESSANDO`.
5. Clicar "Atualizar status" → card atualiza para `AUTORIZADO` (ou aguardar webhook).
6. Links de PDF e XML aparecem quando `AUTORIZADO`.
